A2 · Graph Traversal

目标：新增 walkGraph() 函数——从 seed nodes 出发，1-hop 遍历图，按边 confidence 打分。作为三路搜索的 Graph 路。

文件类型改动内容
query/graph-search.ts 新增 新增 walkGraph(seedIds, graph) 函数：
① 遍历 graph.edges，找所有与 seedIds 有边的节点（1-hop）
② 每条边取 edge.confidence（已存在）
③ 评分公式（对齐 agentmemory）：avgConfidence × (1/pathLength)——pathLength=1
④ 同一个节点被多个 seed 引用 → 取最高分
knowledge/graph.ts 修改 导出 walkGraph 或保持 graph-search.ts 为独立模块（推荐——职责分离）
tests/graph-search.test.ts 新增 测试：单 seed 1-hop、多 seed 去重、空 graph、confidence=0 的边、contradicts vs derived_from 置信度差异 
export function walkGraph(
  seedIds: string[],
  graph: GraphData,
): Map<string, number> {
  const scores = new Map<string, number>();
  const seedSet = new Set(seedIds);

  for (const edge of graph.edges) {
    const fromSeed = seedSet.has(edge.from);
    const toSeed = seedSet.has(edge.to);
    if (!fromSeed && !toSeed) continue;  // 与任何 seed 无关

    const neighborId = fromSeed ? edge.to : edge.from;
    if (seedSet.has(neighborId)) continue; // 跳过 seed 自身

    const confidence = edge.confidence ?? 0.5;
    const score = confidence * (1 / 1); // pathLength=1

    // 取最高分（不是累加——对齐 agentmemory 的 expandFromChunks 逻辑）
    const current = scores.get(neighborId) ?? 0;
    if (score > current) scores.set(neighborId, score);
  }
  return scores;
}

A5 · Daemon 队列消费

目标：pendingAudit 入队后实际触发 runSemanticAudit()，而不是只 push 不消费。

文件类型改动内容
daemon/index.ts 修改 在 onWikiChange 回调中：push queue 后，异步调用 runSemanticAudit(config, { llmJudge }) → 写回 frontmatter。处理完后从队列移除。 
onWikiChange: (wikiPath: string) => {
  log("info", `[daemon] ③ wiki changed: ${wikiPath}`);
  state.queue.pendingAudit.push(wikiPath);
  writeDaemonState(projectRoot, state);

  // 异步消费——不阻塞 watcher
  runSemanticAudit(config, { llmJudge: async (p) => {
    const r = await client.chat({ model: config.model, messages: [{role:"user",content:p}], responseFormat:"json_object", thinkingDisabled:true, maxTokens:4096 });
    return r.content;
  }}).then(result => {
    writeSemanticAuditResults(config, result);
    state.queue.pendingAudit = state.queue.pendingAudit.filter(p => p !== wikiPath);
    writeDaemonState(projectRoot, state);
  }).catch(err => { log("error", `audit failed: ${err.message}`); });
},

验证：daemon 运行时修改一个 wiki 节点 → daemon.log 中出现 audit 日志 → frontmatter 中 auditStatus 更新。
