关键设计改进（v2.0 vs v1.0）：
  • 命题提取是 compile 的前置步骤——LLM 将 chase 文本拆分为原子命题，标注 <!-- prop N --> marker。一次 LLM 调用，结果持久化。
  • 覆盖式 ingest——同一篇材料重编译时，先删掉所有 sourceIds 包含该 source 的旧节点，再写新的。
  • Edge 是编译的自动产出——compile LLM 在输出节点时同时标注 derived_from 和 related 边（confidence 来自 LLM 自评）。
  • 矛盾 + 强化检测在 compile 后自动运行，候选写入 progress.md 等待人类确认。
