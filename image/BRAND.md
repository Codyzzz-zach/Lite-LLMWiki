# LiteWikiagent Brand Identity

> Agent-first Knowledge Compiler · Brand Design System v1.0

---

## Logo Concept

The icon mark uses a **W-shaped compiled node path** — three knowledge nodes connected
by a structural trace, echoing the "wiki" abbreviation while visualizing the core
product metaphor: raw material → compiled structure → auditable wiki.

The top circle represents the single source (the raw file).
The bottom two circles are the compiled outputs (wiki nodes with traceability).
The W-path in between is the compilation trace.

## Brand Colors

| Token | Hex | Usage |
|-------|-----|-------|
| Ink Black | `#0A0A0A` | Primary dark background |
| Zinc 950 | `#18181B` | Secondary dark surface |
| Forest Primary | `#0D5C41` | Brand primary — trust, depth, knowledge |
| Compile Green | `#00D97E` | Accent highlight — active state, success |
| Surface Mint | `#F0FDF4` | Light surface tint |
| Off White | `#F9FAFB` | Light background |

### Semantic Pairings

- Dark UI: `#0A0A0A` bg + `#00D97E` accent + `#fafafa` text
- Light UI: `#F9FAFB` bg + `#0D5C41` primary + `#18181b` text
- Code/terminal: `#0D5C41` bg + `#ffffff` marks

## Typography

- **Display / UI**: Geist (or SF Pro Display as fallback)
- **Code / Labels**: Geist Mono (or SF Mono as fallback)
- **No serif fonts** — this is a developer tool, not an editorial product

---

## Brand Voice

- Precise, not verbose
- Traceable, not magical
- Structural, not decorative

Tagline: **"KNOWLEDGE COMPILER"** (set in Geist Mono, letter-spacing 2px, uppercase)

---

## Do / Don't

| Do | Don't |
|----|-------|
| Use `#0D5C41` as the primary brand color | Use purple/indigo — no AI purple trap |
| Keep icon mark simple and geometric | Add gradients to the icon |
| Use Geist Mono for technical labels | Use Inter or a generic sans |
| Dark background for terminal / README contexts | Use pure black `#000000` |
| Keep `agent` in accent color in wordmark | Make full wordmark one color |
